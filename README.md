# Wildlife Image Classification using Custom CNNs and Transfer Learning

## Overview

This project explores two different approaches to image classification on the **iNaturalist 12K dataset** containing images from 10 biological categories:

* Amphibia
* Animalia
* Arachnida
* Aves
* Fungi
* Insecta
* Mammalia
* Mollusca
* Plantae
* Reptilia

The project is divided into two major parts:

1. **Custom Convolutional Neural Network (CNN) built from scratch**
2. **Transfer Learning using a pre-trained ResNet50 model**

The objective is to compare training a CNN from scratch against leveraging powerful pre-trained representations learned from the ImageNet dataset.

---

# Dataset

The project uses the **iNaturalist 12K** dataset consisting of approximately **10,000 training images** and **2,000 validation/test images** spread across 10 classes.

## Data Processing

### Custom CNN

Images are:

* Resized to a fixed resolution
* Converted to tensors
* Optionally augmented using:

  * Random Crop
  * Random Horizontal Flip
  * Random Rotation

### Transfer Learning

For ResNet50, the preprocessing pipeline follows the transformations expected by the ImageNet pre-trained weights:

* Resize
* Crop
* Tensor Conversion
* ImageNet Normalization

Optional data augmentation is also applied during training.

---

# Part A: Custom CNN from Scratch

## Architecture

The custom CNN is implemented entirely using PyTorch modules without relying on pre-built architectures.

Each convolutional block follows:

Conv2D → BatchNorm (optional) → Activation → MaxPooling

The network supports:

* Variable number of convolution blocks
* Variable number of filters
* Configurable kernel sizes
* Configurable strides and paddings
* Multiple fully connected layers
* Batch Normalization
* Dropout Regularization

The architecture is dynamically built using `nn.ModuleList`.

---

# Part B: Transfer Learning with ResNet50

Training deep CNNs from scratch often requires massive datasets and extensive computational resources.

To overcome this, transfer learning is performed using a **ResNet50 model pre-trained on ImageNet**.

The final classification layer is replaced to support the 10 classes of the iNaturalist dataset.

---

## Fine-Tuning Strategies

Three different transfer learning strategies were explored.

### Strategy 1: Fixed Feature Extractor

All layers are frozen except the final classification layer.

Only the final fully connected layer is trained.

Advantages:

* Fastest training
* Lowest memory consumption
* Useful when dataset size is small

---

### Strategy 2: Partial Fine-Tuning

All layers are frozen except:

* Layer4
* Final Fully Connected Layer

Advantages:

* Learns dataset-specific features
* Lower computational cost than full fine-tuning
* Often provides the best trade-off between accuracy and training time

---

### Strategy 3: Full Fine-Tuning

All parameters of ResNet50 are unfrozen and updated during training.

Advantages:

* Highest learning capacity
* Allows complete adaptation to the target dataset

Disadvantages:

* Highest computational cost
* Increased memory requirements

---

## Hyperparameter Optimization

Weights & Biases Sweeps were used to perform automated hyperparameter search over:

* Learning Rate
* Batch Size
* Weight Decay
* Data Augmentation
* Fine-Tuning Strategy

Bayesian Optimization was used to efficiently explore the search space.
